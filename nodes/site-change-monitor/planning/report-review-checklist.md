# Report review checklist

## Syfte
Bedöma om per-run JSON + HTML-rapporten är tillräckligt bra för riktiga tester och demo.

## Kontrollpunkter
- Är det direkt tydligt vilka URL:er som ändrats?
- Är severity begriplig utan förklaring?
- Räcker summary-texten för att förstå ändringen?
- Behöver man öppna diff-filen ofta, eller bara ibland?
- Är artifact paths användbara i praktiken?
- Känns HTML-rapporten tillräckligt tydlig för att visa en testanvändare?
- Finns det onödigt brus i unchanged/minor-resultat?

## Beslut efter 5–10 körningar
- Behåll rapportformat som det är
- Justera summary/severity
- Lägg till scheduler som nästa slice
- Vänta med fler features
