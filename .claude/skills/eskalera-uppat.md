# Eskalera uppåt

## Eskaleringskedjan

```
Du → Teamleader → Rikard (VD)
```

Hoppa aldrig steg. Om du inte kan lösa det → teamleadern. Teamleadern eskalerar om han inte kan besluta.

## När du eskalerar till Teamleadern

- Uppgiften kräver koordination med minst en till agent
- Du saknar ett MCP-verktyg som behövs (rapportera luckan)
- Uppgiften är tvetydig och du har tolkat den — verifiera tolkningen
- Du har gjort klart din del och nästa steg tillhör en annan agent

## När Teamleadern eskalerar till Rikard

- Beslut som ändrar arkitekturen (ny datakälla, ny integration)
- Nyrekrytering (ny agent ska godkännas)
- Kostnadsprojektionen är röd och osäkerhet finns om körningen är värd det
- Inget i teamet kan hantera uppgiften — ny kompetens krävs

## Format för eskalering

```
ESKALERAR TILL: [teamleader | Rikard]
ANLEDNING: [varför du inte kan hantera detta ensam]
KONTEXT: [vad du redan gjort/vet]
BESLUT SOM KRÄVS: [exakt vad som behöver beslutas]
TIDSKÄNSLIGHET: [kan vänta | bör hanteras denna session | akut]
```

## Vad du INTE eskalerar

- Rutinuppgifter som matchar din roll
- Tekniska beslut inom din kompetens
- Frågor du kan svara på med tillgängliga verktyg
- Frågor som bara kräver att du läser mer data
